# paper-distiller v1.5 — Claude Code-style UX + Agent Control

**Status:** draft
**Goal:** Borrow six design patterns from Claude Code's source to improve paper-distiller's interaction experience and LLM-control logic. Python only — no rewrites.
**Anchor commit:** `5d4a5c8` (v1.4.0 release)

---

## Motivation

v1.4 made paper-distiller a conversational agent: user types natural language, an internal LLM decides which of 5 tools to invoke. The basics work, but the UX is thin:

- Agent must guess parameters (`research duration="2h", max_papers=20`) with no way to ask the user
- Slash-command muscle memory is gone — every `/cost` or `/show` has to round-trip through the LLM
- LLM responses arrive all-at-once; long `research()` runs feel like hangs
- Ctrl-C kills the whole REPL, not just the current tool
- No session-wide cost view; user can't tell when they're about to burn ¥30
- Expensive operations (`research`, `ask` with high `max_cost_cny`) execute silently with no preview

Claude Code's source code has solid answers to all six. We adopt them in Python.

---

## Scope (Standard bundle)

1. **`ask_user` tool** — LLM-callable interrupt that surfaces a multiple-choice question to the user mid-task
2. **Slash commands** alongside natural language (`/clear /cost /help /show /history /exit /auto`)
3. **Streaming text output** for the assistant's natural-language replies
4. **Session-wide cost display** (tokens + estimated ¥) after each turn and via `/cost`
5. **Plan mode** — for tools whose `max_cost_cny > threshold` (default ¥10), print a structured preview with a 5-second countdown before execution
6. **Ctrl-C abort** — first press cancels the currently running tool; second press inside a short window exits the REPL

## Design

### 1. `ask_user` tool

A 6th entry in `TOOL_SCHEMAS` and `TOOL_FUNCTIONS`. Schema mirrors Claude Code's `AskUserQuestionTool` but trimmed:

```python
{
    "type": "function",
    "function": {
        "name": "ask_user",
        "description": (
            "Ask the user a multiple-choice question and wait for their reply. "
            "Use this when you need a decision the user should make: budget "
            "size, which paper to distill from a search result, whether to "
            "proceed with a costly research run, etc. Do NOT use for trivial "
            "confirmations the agent can decide itself."
        ),
        "parameters": {
            "type": "object",
            "properties": {
                "question": {"type": "string"},
                "header": {
                    "type": "string",
                    "description": "Short chip label (<=12 chars)."
                },
                "options": {
                    "type": "array",
                    "items": {
                        "type": "object",
                        "properties": {
                            "label": {"type": "string"},
                            "description": {"type": "string"}
                        },
                        "required": ["label", "description"]
                    },
                    "minItems": 2,
                    "maxItems": 4
                },
                "multi_select": {"type": "boolean", "default": False}
            },
            "required": ["question", "options"]
        }
    }
}
```

**Wrapper behavior** (`tool_ask_user(question, options, header="?", multi_select=False, *, vault_path)`):
- Print the question, header chip, and options (numbered 1..N) via `rich`
- Read input via `input()`; allow `1`, `1,3`, `q` to cancel
- Return `{"selected": ["label of #1", ...], "cancelled": False}` or `{"cancelled": True}`
- `vault_path` is ignored but kept for `execute_tool` compatibility

System prompt addition tells the LLM when to call this tool vs decide autonomously. Default behavior remains "decide autonomously"; `ask_user` is for genuine ambiguity.

### 2. Slash commands

Inside `AgentLoop.run()`, before sending to the LLM, check `line.startswith("/")`. If so, route to a slash handler — these never reach the LLM.

| Command | Effect |
|---|---|
| `/clear` | Reset message history to just the system prompt. Print `(history cleared)` |
| `/cost` | Print tokens in/out + estimated ¥ for this session |
| `/help` | List all slash commands + a one-liner per tool from `TOOL_SCHEMAS` |
| `/show <slug> [category]` | Read vault entry directly (no LLM round-trip). Renders body as markdown. |
| `/history` | List 10 most-recent vault entries with slug + title + updated date |
| `/exit` (and `/quit`) | Exit the REPL |
| `/auto` | Toggle "auto mode" — skips Plan-mode countdown and budget previews. Default OFF. |

Implementation: a `dict[str, Callable[[AgentLoop, list[str]], None]]` map in `chat/slash_commands.py`. Each handler takes the AgentLoop instance (for state access) and the post-command tokens. Errors print `[red]unknown command[/red]` and a `/help` hint.

### 3. Streaming text output

Extend `LLMClient` with `complete_with_tools_stream(messages, tools, temperature=0.5) -> Iterator[StreamChunk]`. Use Aliyun Bailian's SSE streaming endpoint:

```python
body = {..., "stream": True}
with self._client.stream("POST", url, json=body, headers=...) as r:
    for sse_line in r.iter_lines():
        if not sse_line.startswith("data: "): continue
        data = json.loads(sse_line[6:])
        # ... yield chunks
```

`StreamChunk` is a dataclass:
```python
@dataclass
class StreamChunk:
    text_delta: str = ""
    tool_call_id: str | None = None
    tool_name_delta: str = ""
    tool_arg_delta: str = ""
    finish_reason: str = ""
```

Tool calls in OpenAI streaming format arrive in pieces — `function.name` may come in chunk 0, then `function.arguments` in chunks 1..N as a JSON string. The AgentLoop accumulates these into complete `ToolCall` objects before dispatching.

**AgentLoop integration:** `AgentLoop.send()` becomes a generator that yields display fragments. The interactive `run()` consumes via `rich.live.Live` for text and pre-renders tool-call markers as soon as the tool name lands.

Streaming is **always on** in v1.5. There is no opt-out env — if a provider doesn't support streaming, the fallback path is to read the full response and emit one StreamChunk with everything. Existing `complete_with_tools` stays for tests that don't want streams.

### 4. Session-wide cost display

New module `llm/pricing.py`:
```python
PRICING_PER_M_TOKENS_CNY = {
    "qwen-plus": {"in": 0.8, "out": 2.0},
    "qwen-turbo": {"in": 0.3, "out": 0.6},
    "qwen-max": {"in": 20.0, "out": 60.0},
    "deepseek-chat": {"in": 0.5, "out": 1.5},
    # ... (extend as we encounter new models)
}

def estimate_cost_cny(model: str, tokens_in: int, tokens_out: int) -> float:
    rate = PRICING_PER_M_TOKENS_CNY.get(model, {"in": 1.0, "out": 3.0})
    return tokens_in / 1e6 * rate["in"] + tokens_out / 1e6 * rate["out"]
```

Env overrides: `PD_PRICE_IN_CNY_PER_M`, `PD_PRICE_OUT_CNY_PER_M`. Unknown models log a one-line warning and use the conservative default.

`LLMClient` exposes `estimated_cost_cny` (read-only property computed from `total_tokens_in/out + model`).

Display after each turn (after the streaming reply ends):
```
[dim]tokens in/out: 4231 / 1820  ·  ¥0.0072  ·  session total: ¥0.041[/dim]
```

`/cost` prints a fuller breakdown.

### 5. Plan mode

For tool calls where the tool is `research` OR where `arguments.get("max_cost_cny", 0) > threshold`, the AgentLoop intercepts BEFORE executing:

1. Compose a "plan card" — the tool name, all arguments, and the estimated cost ceiling
2. Print it with a 5-second countdown (`Enter to proceed, Ctrl-C to cancel, /auto to skip future cards`)
3. On Enter or timeout → proceed with the tool
4. On Ctrl-C → respond to the LLM with `{"cancelled": True, "reason": "user cancelled plan"}` and continue the conversation

Threshold: `PD_PLAN_THRESHOLD_CNY` env, default `10.0`. When `/auto` toggle is on, the threshold is effectively `+inf` (everything proceeds without preview).

Plan-card display (rich.Panel):
```
┌─ Plan: research ─────────────────────── ¥15 budget ─┐
│ question:      "扩散模型最新综述"                    │
│ duration:      2h                                    │
│ max_papers:    20                                    │
│ max_cost_cny:  15.0                                  │
│                                                      │
│ Press Enter to proceed (auto-proceed in 5s)         │
│ Ctrl-C to cancel · /auto to disable previews        │
└──────────────────────────────────────────────────────┘
```

Only `research` and `ask` will commonly trigger this with default args. `distill_by_id` for 20+ papers might too — we use estimated cost: `tool_distill_by_id` estimates ¥0.2 per paper.

Per-tool cost-estimate functions in `chat/cost_estimator.py`:
```python
def estimate_tool_cost_cny(name: str, arguments: dict) -> float:
    if name == "research":
        return float(arguments.get("max_cost_cny", 15.0))
    if name == "ask":
        return float(arguments.get("max_cost_cny", 5.0))
    if name == "distill_by_id":
        return 0.2 * len(arguments.get("ids", []) or [])
    if name == "search":
        return 0.05  # ranker LLM call
    return 0.0
```

### 6. Ctrl-C abort

A SIGINT handler installed by `AgentLoop.run()`:

- **State 1 — idle (no tool running):** ignore SIGINT (Python default would raise KeyboardInterrupt; we suppress for state cleanliness during `input()`)
- **State 2 — tool running:** set `self._abort_event` (a `threading.Event`); tool wrappers running `asyncio.run(orch.run())` are wrapped in `try/except KeyboardInterrupt → return {"cancelled": True, "by": "user"}`. The cancellation propagates because Python's default SIGINT raises KeyboardInterrupt in the main thread, which interrupts whatever `asyncio.run` is doing.
- **State 3 — second SIGINT within 1s:** exit the REPL (legacy Ctrl-C-twice behavior).

The asyncio orchestrator does NOT need cooperative cancellation for this to work — `asyncio.run` catches KeyboardInterrupt and cancels all running tasks before re-raising. The wrapper just needs to return a clean cancelled-result dict instead of letting KeyboardInterrupt escape.

After a cancel: the loop posts a synthetic `tool` message with `{"cancelled": True, "by": "user"}` and continues with the next LLM call. The LLM sees the cancellation and adapts its reply.

## Cross-cutting decisions

**Default-on behaviors:** streaming is on by default. Plan mode is on by default with threshold ¥10. Slash commands are always available. Cost display is always on.

**Toggleable:** `/auto` disables plan-mode previews for the rest of the session.

**Env overrides:**
- `PD_PLAN_THRESHOLD_CNY` (default `10.0`)
- `PD_PRICE_IN_CNY_PER_M` / `PD_PRICE_OUT_CNY_PER_M` (override pricing table)
- `PD_STREAM=0` (force non-streaming if a provider misbehaves)

**Backward compatibility:**
- All existing tools (search/distill_by_id/show/ask/research) unchanged in behavior; `ask_user` is added as a 6th
- All existing single-shot CLI subcommands (distill/browse/ask/resume/research/legacy-repl) unchanged
- The `complete_with_tools` non-streaming method stays; new code uses `complete_with_tools_stream`

## Open questions for review

1. **Plan-mode auto-proceed timer** — is 5s right? Too fast for reading, too slow for muscle memory?
2. **`/auto` scope** — session-only, or also persisted to a config file? Proposed: session-only. User can set `PD_AUTO=1` env for sticky.
3. **`ask_user` while streaming** — if the LLM emits an `ask_user` call mid-stream, we need to pause the stream and prompt. Proposed: same UX as plan mode — block on input(), then continue conversation with the user's selection.
4. **Cost display granularity** — show ¥ to 4 decimals, or round to ¥0.01 for short turns? Proposed: 4 decimals (¥0.0072 is meaningful at qwen scale).
